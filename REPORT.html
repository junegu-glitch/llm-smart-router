Project summary: I built llm-smart-router, a CLI-first AI workflow copilot that routes work to the best available LLM and can coordinate small multi-model agent teams.

1. Summary

This project explores a simple idea: one model is rarely best at everything. Instead of forcing every prompt through a single provider, llm-smart-router classifies the task, selects a better-fit model, and can scale up to a team workflow when the task is larger or more specialized.

2. What Project 1 Was

My project-1 was plateprep, a Python CLI for preprocessing 96-well absorbance plate-reader CSV exports. It validated four wavelength files, merged them into one analysis-ready table, and printed summary statistics. It was useful and practical, but intentionally narrow in domain and scope.

3. What Is Different From Project 1

Category Project 1: plateprep New project: llm-smart-router
Primary problem Lab CSV preprocessing AI workflow orchestration
Audience Lab users working with plate-reader exports Developers and technical users working across LLM providers
Interface Simple Python CLI with 3 commands TypeScript CLI with routing, team mode, presets, and session review
Input type Structured CSV files Natural-language prompts, git diffs, and selected files
Output type Clean CSV and statistics Routed LLM responses, team reports, and review output

The key improvement I define here is generality plus orchestration. Project-1 automated one recurring data-preparation task. This project automates a wider class of technical work and makes model specialization a first-class design choice.

4. Main Features

  • Task classification into categories such as coding, writing, analysis, math/reasoning, and large-document work
  • Model selection with ranked fallback across providers
  • Team mode with a leader, parallel teammates, and synthesized final output
  • Preset workflows for code review, debugging, refactoring, and explanation
  • Git-aware context via --git-diff and file selection
  • Session review for past team runs

5. Example Usage

Single auto-routed prompt

smart-router "Implement quicksort in Python"

Expected behavior: classify as coding and choose a strong coding model.

Team workflow

smart-router team run "Compare React vs Vue for a small SaaS dashboard"

Expected behavior: plan several specialist roles, run them in parallel, and synthesize one final answer.

Code review with git context

smart-router team run --preset code-review --git-diff "Review my current changes"

Expected behavior: run a code-review-oriented team using actual repository context.

7. Reflection

The main lesson I carried forward from project-1 was that a CLI becomes valuable when it compresses a repeated manual workflow into one reliable command. I kept that lesson but changed the domain completely. Instead of automating one lab preprocessing step, I built a more ambitious developer tool that automates LLM selection and multi-model coordination.

This project also demanded broader system design than plateprep: routing logic, provider abstraction, multi-agent orchestration, session handling, and a larger test surface. That makes it a better demonstration of what I learned after project-1.
