## Design System (in gen_pptx.js)

```javascript
const C = {
  navy:    "1a1a2e",   // dark navy — dark slide bg, headings
  navy2:   "16213e",   // slightly lighter navy
  indigo:  "4f46e5",   // primary accent (top bar, separators, badges)
  indigoL: "eef2ff",   // light indigo for badges
  white:   "FFFFFF",
  gray50:  "F9FAFB",
  gray100: "F3F4F6",
  gray200: "E5E7EB",
  gray400: "9CA3AF",
  gray600: "4B5563",
  gray700: "374151",   // body text
  code_bg: "1e1e1e",   // dark code block bg
  code_fg: "D4D4D4",   // code text color
};
```

**Layout**: LAYOUT_16x9 — 10" × 5.625"

**Slide patterns**:
- `darkSlide()` — navy bg, used for title/demo/closing slides
- `lightSlide()` — white bg, used for content slides
- `topBar(slide)` — indigo top accent bar (0.06" tall), all light slides
- `darkTopBar(slide)` — slightly lighter purple bar, all dark slides
- `slideTitle(slide, text)` — navy bold title at y=0.25, indigo separator line underneath
- `codeBlock(slide, text, x, y, w, h)` — dark bg + Consolas monospace text
- `badge(slide, text, x, y)` — small indigo rounded badge

**Fonts**: Calibri (body/headings), Consolas (code/monospace)

**pptxgenjs rules to follow**:
- NEVER use `#` prefix in hex colors (causes file corruption)
- NEVER encode opacity in color strings (use `opacity` property instead)
- NEVER reuse options objects across addShape/addText calls (pptxgenjs mutates them)
- Use `bullet: true` in text options, never unicode "•"
- Use `breakLine: true` between array text items
